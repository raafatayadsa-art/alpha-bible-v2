# Alpha Bible App Rules

## Project Identity
- This is the active Alpha Bible mobile app.
- Do not rebuild.
- Do not create a new project.

## Brand and Product Protection
- Do not change logo.
- Do not change bottom navigation.
- Do not change approved Alpha DNA.
- All changes must be incremental improvements only.

## Platform Targets
- iPhone
- Android phones
- iPad
- Android tablets
- Large tablets
- Future PWA support

## Mandatory Product Quality Standards
The app must always be:
- mobile-first
- touch-first
- tablet responsive
- safe-area aware
- optimized for smooth scrolling and animations
- optimized for mobile performance

## Architecture Preservation
Preserve current:
- Vite
- React
- TanStack Router
- Tailwind v4
architecture.

## UI and Scope Control
- Do not touch UI unless explicitly requested.
- Do not modify app UI.
- Documentation/rules only unless explicitly instructed otherwise.

## Required Change Communication
Before editing, always explain:
- files changed
- why changes are needed
- possible risks

## High-Risk Areas (Confirmation Required)
Never modify the following without explicit confirmation:
- splash screen
- routing
- layouts
- navigation structure
