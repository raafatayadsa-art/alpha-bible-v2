# ALPHA-100 — Saint Community Gallery & Dynamic Saint Card System

**Date:** 2026-06-18  
**Scope:** Database schema, API layer, UI wiring, admin approval sync  
**Build:** PASS (`npm run build`)

---

## Executive Summary

Implemented the core **ALPHA-100** saint community gallery system: users upload images from a saint detail page (auto-linked via `saint_id`), submissions enter review, Alpha Manager receives `saint_image` approvals, approved images populate auto-generated per-saint albums, and the **Saint of the Day** card supports crossfade image cycling, share picker, likes, and album access. Profile **مساهماتي → الصور** tracks submission status.

## Warnings

1. **Migration not applied remotely** — Run `supabase db push` or apply `20250618180000_saint_community_gallery.sql` on the target Supabase project before live uploads work.
2. **Auth required for upload** — `submitSaintGalleryImage` returns null without session; guest users see an error in the upload sheet.
3. **Home hero card** — Full dynamic controls (cycle/share picker) are on Synaxarium index; home stack only resolves community featured image URL.
4. **Contests** — Album shows top-liked strip (monthly winner label); no separate contest tables or seasonal campaigns yet.
5. **Likes on dynamic card** — Local UI toggle only on hero card; album likes use API but hero like is not persisted to DB yet.
