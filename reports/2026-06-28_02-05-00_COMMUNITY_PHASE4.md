# Community Platform Phase 4 — Auto Agpeya, Profile Activity, Groups Filters

**Date:** 2026-06-28  
**Project:** Alpha Bible  
**Scope:** Agpeya auto-activity, feed refresh on friend sync, profile activity tab, groups tab filters

---

## Executive Summary

Phase 4 completes the auto-activity engine for Agpeya, refreshes the community feed when friends sync from Supabase, populates the profile **نشاط** tab with community moments, and makes groups discovery tabs functional (الكل · مجموعاتي · كنيستي · مقترحة). Build passes.

---

## Findings

### Auto Agpeya activity

- `maybeEmitAgpeyaActivity()` in `community-auto-activity.ts` fires when scroll progress crosses **92%** (once per prayer, deduped).
- Hooked from `agpeya.$prayerId.tsx` scroll handler alongside position save.
- Moments tagged `source: auto_agpeya` with Arabic summary e.g. «أتم صلاة باكر من الأجبية».
- `CommunityAgpeyaPayload` extended with `auto` + `activitySummary`.

### Feed refresh on friend sync

- `syncCommunityFriendsFromRemote()` now calls `syncCommunityFeed()` after merge.
- `CommunityScreen` re-syncs when `friendUserIds` change.

### Profile activity tab

- New `ProfileCommunityActivitySection.tsx` on profile tab **نشاط** when community module is on.
- Lists up to 8 user moments via `useCommunityUserMoments` + `CommunityFriendActivityItem`.

### Groups screen filters

- Tabs are interactive buttons with real filter logic:
  - **الكل** — all group-type Connect channels
  - **مجموعاتي** — channels where current user is a member
  - **كنيستي** — favorite / church-named channels
  - **مقترحة** — channels user has not joined
- Shows «عضو» badge on joined groups.

### Minor fix

- `CommunityFriendActivityItem` agpeya deep links now route to `/agpeya/$prayerId` correctly.

### Build

- `npm run build` — **PASS** (exit 0).

- Profile activity tab shows local + merged remote moments; may lag until sync completes.
