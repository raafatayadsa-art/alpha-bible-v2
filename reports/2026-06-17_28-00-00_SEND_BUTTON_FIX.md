# Alpha Connect Send Button Fix

**Date:** 2026-06-17  
**Overall Status:** PASS

---

## Executive Summary

Send failed because the chat thread opened before Supabase auth was ready: `getAuthUserSync()` returned null, `conversationId` was never set, and send stopped before the DB insert. Fixed by waiting for auth, retrying on auth load, lazy conversation open on send, and using live `getAuthUserId()` for inserts.

---

## Root Cause

1. **`useAlphaConnectThread`** used `getAuthUserSync()` only at mount — if chat opened before `AuthBootstrap` finished, init aborted permanently (no retry).
2. **`sendText`** also used sync cache — silent `false` when cache empty even if Supabase session existed.
3. **`sendMessageToThread`** blocked send when `conversationId` was null instead of attempting open.

---

## Fix Applied

| File | Change |
|------|--------|
| `useAlphaConnectThread.ts` | `waitForAuthUserId()` + `subscribeAuthContext` retry; `ensureConversationId()` on send; `getAuthUserId()` for sender; surface Supabase errors |
| `AlphaChatScreen.tsx` | Removed hard block on missing `conversationId`; delegate open to `sendText` |
| `RUN_ALPHA_CONNECT_DIRECT_MESSAGING.sql` | **NEW** — manual SQL if RPC missing in Supabase |
